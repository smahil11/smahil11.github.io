Project Games: Metacritic Video Games data and recommender

March 19, 2021

In this project, I collect data from Metacritic to do some basic EDA and build a content-based recommender system. The final recommender is deployed using Dash, and all coding is done in Python.

Scraping games info from Metacritic

The scraper is found in "scraper.ipynb" and requires requests, bs4.BeautifulSoup, re, and pandas.

The final dataset is in the file "games-data.csv".

The dataset

We want to get a dataframe with all games with columns:

  • name: The name of the game
  • platform: Platform it was released
  • r-date: Date it was released
  • score: Average score given by critics (metascore)
  • user score: Average score given by users on the website
  • developer: Game developer
  • genre: Genre of the game (can be multiple)
  • players: Number of players (some games don’t have this information)
  • critics: Number of critics reviewing the game
  • users: Number of Metacritic users that reviewed the game

All data was collected on November 10th, 2020.

Steps used in scraper:

  1. Create a dictionary "pages" that will contain the DataFrame objects from all pages. Each entry is a pandas DataFrame with data from the games on each site page. There are currently 180 pages of rated games.
  2. For each page, create a dictionary "data_page" of empty lists to be filled with the data from each game. As each page displays 100 games, each of these lists should contain 100 elements (except for the last page).
  3. Use requests to get into the URL of each page and BeautifulSoup to parse the HTML file.
  4. Loop through all games on each page and scrape the relevant data. Note that ‘developer’, ‘genre’, ‘players’, ‘critics’, and ‘users’ are found on different URLs, so we need to fetch these for each game. This URL for each game is inside an "a" tag with a "title" class.
  5. There are a couple of if's in the scraper to ensure None objects get dealt with (some games don’t have a number of players' information, for example; some games have no user reviews, given it is not yet released, and a few others).
  6. After all data is collected (and it takes a few hours - a bit more than 15 in my laptop), all dataframes in the "pages" dictionary are concatenated to create a single one with all game data.
  7. The dataframe is exported to a CSV file.
